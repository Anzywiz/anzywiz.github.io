Roadmap

Improvement opportunities

Fields that could be enriched in a future scraper iteration.

Easy
material_info
Material details (e.g. "18K gold", "platinum") are visible on product pages — a single extra selector would capture them reliably.
Easy
specifications
Technical specs (dimensions, weight, stone count) appear in a structured accordion on each PDP — parseable without extra page loads.
Medium
additional_tags (10.4% → 100%)
Currently sparse. Tag data is surfaced in breadcrumb and filter facets — requires a facet crawl pass to fill completely.
Medium
size_info (31% → 80%+)
Size options live in a JS-rendered size picker — a Playwright/Selenium variant or API intercept would capture all options per product.
Medium
full_price / price (85% → 100%)
~15% of products have no displayed price on cartier.com ("price on request" items). These could be enriched via a Cartier boutique price lookup.
Hard
store_id / store_name / store_address
Cartier's boutique stock-check requires a user session and a geo-query API call — feasible but adds significant pipeline complexity.
